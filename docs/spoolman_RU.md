# Обязательная предварительная настройка Spoolman

[English](spoolman.md)

## Важно

**AD5X IFS Plugin for Spoolman не заменяет Spoolman и не устанавливает его.**

Перед установкой плагина уже должны быть выполнены все условия:

1. Spoolman установлен и запущен на отдельном устройстве или сервере;
2. принтер по сети видит этот сервер;
3. Moonraker настроен на подключение к Spoolman;
4. Moonraker показывает `spoolman_connected: true`.

Spoolman можно разместить, например, на:

- домашнем сервере или NAS;
- компьютере, который постоянно включён;
- Raspberry Pi или другом одноплатном компьютере;
- VPS, доступном принтеру по сети.

## 1. Узнайте адрес Spoolman

После установки Spoolman его адрес обычно выглядит так:

```text
http://IP_СЕРВЕРА:7912
```

Пример:

```text
http://192.168.1.50:7912
```

Откройте этот адрес в браузере с компьютера или телефона. Интерфейс Spoolman должен загрузиться.

## 2. Добавьте Spoolman в Moonraker

В конфигурации `moonraker.conf` должна присутствовать секция:

```ini
[spoolman]
server: http://IP_СЕРВЕРА_SPOOLMAN:7912
```

Пример:

```ini
[spoolman]
server: http://192.168.1.50:7912
```

Не добавляйте вторую секцию `[spoolman]`, если она уже существует. Исправьте адрес в существующей секции.

## 3. Перезапустите Moonraker

После изменения `moonraker.conf` перезапустите **только Moonraker** средствами Z-Mod или Fluidd.

## 4. Проверьте соединение по SSH

Подключитесь к принтеру как `root` и выполните:

```sh
wget -qO- http://127.0.0.1:7125/server/spoolman/status
```

Исправное подключение должно содержать:

```json
"spoolman_connected": true
```

Если возвращается `false`, ошибка или пустой ответ, сначала исправьте подключение Moonraker к Spoolman. Установщик AD5X IFS Plugin остановится и не продолжит установку при неработающем Spoolman.

## Частые причины отсутствия соединения

- неверный IP-адрес или порт Spoolman;
- Spoolman запущен только на `127.0.0.1` своего сервера;
- порт `7912` закрыт firewall;
- принтер и сервер находятся в разных сетях без маршрута между ними;
- сервер Spoolman выключен;
- в `moonraker.conf` создано две секции `[spoolman]`;
- Moonraker не был перезапущен после изменения конфигурации.

После того как проверка возвращает `spoolman_connected: true`, переходите к [установке плагина](installation_RU.md).
